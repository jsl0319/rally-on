# RallyOn — 프로젝트 브리프

> 테린이를 위한 테니스 매칭 웹앱. 마지막 갱신: 2026-07-14

## 이 문서의 목적

이 문서는 RallyOn의 **현재 제품 결정의 기준점**이다. 타깃, MVP 범위, 사용자 흐름, 운영 원칙, 지표처럼 팀이 구현·운영에 따라야 할 내용을 기록한다. 외부 자료의 출처, 경쟁사 관찰, 근거 강도와 미검증 가설의 상세는 [웹 리서치 노트](docs/web-research.md)에서 관리한다.

## 1. 제품 한 줄 정의

**RallyOn은 20~30대·구력 3년 이하 테린이가 비슷한 수준과 운동 목적의 사람을 찾아, 랠리 연습부터 부담 없는 게임까지 실제 코트에서 즐길 수 있게 하는 테니스 매칭 웹앱이다.**

단순한 '테니스 친구 찾기'가 아니다. `실력 불확실성 + 코트 확보 + 일정 조율 + 낯선 사람 신뢰`의 문제를 한 흐름에서 줄이고, 연습과 게임 사이의 진입 장벽을 낮추는 서비스다.

## 2. 타깃과 문제

### 핵심 타깃

- 20~30대, 테니스 구력 3년 이하
- 레슨은 받았지만 정기적으로 함께 칠 파트너가 없는 사람
- 기존 동호회의 실력 중심 분위기, 고정 멤버십, 회식 문화가 부담스러운 사람
- 랠리 연습도 하고 싶고, 준비되면 부담 없이 게임도 해보고 싶은 사람

### 해결할 사용자 문제

| 사용자 생각 | RallyOn의 답 |
| --- | --- |
| 내 실력이 어느 정도인지 모르겠다 | 구력 대신 랠리·서브·게임 경험으로 수준을 표현한다. |
| 못 치면 민폐일까 봐 걱정된다 | '랠리 연습', '게임 입문', '초보 게임'의 기대 수준을 분명히 구분한다. |
| 코트 잡기가 너무 어렵다 | 코트가 확보된 모임을 우선 노출하고, 코트 없는 사용자는 대기 매칭한다. |
| 낯선 사람이 노쇼하면 어쩌지 | 신뢰도, 출석 확인, 취소 규칙, 후기 시스템으로 안전을 만든다. |
| 동호회 관계가 부담스럽다 | 가입·회식 의무 없는 가벼운 1회성 운동 모임을 기본으로 한다. |

## 3. 제품 원칙

1. **구력보다 실제 플레이 경험.** 3년 이하도 실력 차가 크므로 구력은 보조 정보만 쓴다.
2. **연습과 게임을 모두 안전하게.** 랠리는 기본기를 익히는 선택지이고, 게임은 부담 없이 도전할 수 있는 다음 선택지다.
3. **매칭보다 실제 플레이.** 연습이든 게임이든 가입자 수가 아니라 실제 출석 세션 수를 가장 중요하게 본다.
4. **코트와 사람을 함께 해결.** 테니스 매칭은 코트 확보 여부가 결과를 좌우한다.
5. **심리적으로 안전한 언어.** 등급·순위·고수 중심 표현을 피하고, 목적과 기대 수준을 명확히 한다.
6. **운동만 하고 헤어져도 정상.** 친목과 회식은 선택이며 서비스가 강요하지 않는다.

## 4. 확정된 온보딩

온보딩의 목표는 정교한 데이터 수집이 아니라 **30초 안에 사용자가 첫 모임을 볼 수 있게 하는 것**이다.

### 받는 정보 — 4개

1. **활동 지역**: 현재 위치 기반 또는 선호 지역 1~3개
2. **테니스 레벨**
   - 공을 거의 못 넘겨요
   - 랠리 3회 정도 가능해요
   - 랠리 5회 이상 가능해요
   - 게임을 조금 해봤어요
3. **오늘의 목적**: 랠리 연습 / 서브 연습 / 게임 입문 / 초보 게임 / 복식 체험
4. **참여 선호**: 1:1 / 2:2 / 상관없음
   - 성별 선호는 필요할 때만 선택값으로 제공

### 온보딩에서 받지 않는 정보 — 확정

- **이동 가능 시간**: 모임 탐색 필터에서 매번 설정한다.
- **코트 보유 여부**: 모임을 만드는 단계에서만 묻는다.

## 5. 핵심 기능과 MVP 범위

### A. 모임 탐색 — MVP 필수

- 오늘 / 내 레벨 / 내 지역 기준의 모임 피드
- 필터: 날짜, 지역, 랠리 레벨, 운동 목적, 1:1·2:2, 코트 확보됨
- 모임 카드 필수 정보
  - 날짜·시간·운동 시간
  - 코트명, 실내·실외
  - `코트 예약 완료` 여부
  - 운동 목적 (`랠리 연습`, `게임 입문`, `초보 게임`, `복식 체험` 등)
  - 요구 레벨, 현재 인원·정원
  - 1인 예상 비용·정산 방식
  - 호스트 신뢰도

### B. 모임 만들기 — MVP 필수

첫 단계에서만 아래 두 경로를 제공한다.

- **코트를 예약했어요**: 코트·시간·비용을 입력하고 인원을 모집한다.
- **코트를 아직 못 구했어요**: 지역·날짜·시간대·레벨이 맞는 사용자와 대기 매칭한다.

### C. 참여·운영 — MVP 필수

- 참여 신청과 호스트 승인(처음에는 승인형 권장)
- 매칭 뒤 모임 전용 채팅
- 고정 공지: 코트, 집결 위치, 비용, 준비물, 우천 규칙
- 참여 전 재확인
- 종료 후 간단 후기

### D. 신뢰와 안전 — MVP 필수

- 휴대전화 본인인증
- 취소 마감·무단 불참 정책
- 신고·차단
- 후기: 시간 준수 / 레벨 일치 / 배려 / 재매칭 의향
- 외모·신체 평가 후기 금지
- 실제 거주지와 실시간 위치는 공개하지 않음

### MVP에서 제외

- 모든 테니스장 실시간 예약 연동과 자동 예약
- 복잡한 AI 추천 모델
- 실력 랭킹과 승패 기반 리더보드
- 대회 운영, 레슨 예약, 장비 커머스
- 유료 구독

## 6. 주요 사용자 흐름

```text
가입 → 30초 온보딩 → 내 지역·레벨 모임 탐색
                          ↓
                 모임 선택 또는 직접 생성
                          ↓
              신청 → 승인 → 모임 채팅·참석 확인
                          ↓
                    코트에서 실제 플레이
                          ↓
                  후기 → 다음 모임 추천
```

## 7. 모임 템플릿

초기에는 자유 작성보다 표준 템플릿을 우선 노출한다.

- **첫 랠리**: 60분, 공 3회 이어 치기를 목표로 하는 기초 연습
- **랠리 온**: 90분, 랠리 3~5회 가능한 사람
- **서브 첫 연습**: 서브 경험이 적은 사람끼리 60분
- **게임 첫 도전**: 노애드 4게임, 규칙을 함께 익힘
- **초보 게임**: 비슷한 레벨끼리 가볍게 단식 또는 복식 게임
- **초보 복식**: 4명, 포지션과 로테이션을 익히며 게임까지 진행

## 8. 매칭 기준

초기에는 설명 가능한 규칙 기반 추천으로 시작한다.

`레벨 적합도 > 운동 목적 일치 > 날짜·지역 > 인원 구성 > 신뢰도`

- 레벨 차이는 기본적으로 ±1단계
- 연습·게임 목적은 기본적으로 분리하되, '연습 후 미니 게임'처럼 명시된 혼합 모임은 별도 유형으로 추천
- 코트 확보 모임을 우선 노출
- 첫 참여자에게는 소규모·입문 모임을 우선 제안
- 신뢰도는 영구 배제보다 재확인·승인 절차 강화에 사용

## 9. UX·카피 가이드

### 말투

- 비경쟁적, 구체적, 안심시키는 표현 사용
- '초보라서 괜찮아요'보다 '오늘은 랠리 60분 후 노애드 2게임을 해요'처럼 기대치를 명확히 제시
- '고수/하수'와 같은 위계 언어를 사용하지 않음

### 예시 카피

- 오늘, 부담 없이 연습하고 게임할 테린이를 찾아보세요.
- 랠리부터, 준비되면 첫 게임까지.
- 공이 자주 끊겨도 괜찮아요. 내 수준의 모임에서 시작해요.
- 코트는 잡았고, 이제 같이 칠 사람만 찾으면 돼요.
- 첫 참여 환영 · 랠리 연습 · 초보 게임 가능

## 10. 출시 전략

- 전국 동시 출시 대신 **서울 내 코트 밀집 생활권 1~2곳**에서 시작한다.
- 초기에는 코트를 잡을 수 있는 호스트 30~50명을 먼저 확보한다.
- 매주 정해진 'RallyOn 플레이 데이'를 운영해 랠리 연습과 초보 게임을 모두 경험하게 한다.
- 레슨장, 테니스 인스타그램, 대학·직장 동아리가 초기 공급 확보 채널이다.
- 첫 목표는 다운로드가 아니라 '매주 실제로 열리는 안전한 10개 모임'이다.

## 11. 지표

### 북극성 지표

**주간 실제 플레이 세션 수**

### 초기 핵심 지표

- 매칭 요청 대비 실제 만남 비율
- 출석률과 노쇼율
- 첫 참여 후 30일 내 재참여율
- '레벨이 적절했다' 후기 비율
- 호스트 1인당 월간 개최 모임 수

## 12. 검증할 가설

출시 전에 타깃 사용자 15~20명 인터뷰와 수동 매칭 30건을 운영한다.

1. 레벨 불일치가 매칭 이탈의 가장 큰 원인인가?
2. 코트 미확보가 매칭 미성사의 가장 큰 원인인가?
3. 첫 모임은 1:1, 2:2, 4인 연습 중 무엇이 출석·재참여가 높은가?
4. 랠리 연습, 게임 입문, 연습 후 미니 게임 중 어떤 유형이 초보자 재참여를 높이는가?
5. 어느 취소 정책이 노쇼를 줄이면서 신규 가입 장벽을 낮추는가?

## 13. 다음 프로젝트에서 바로 할 일

1. 핵심 화면 3개(온보딩, 모임 탐색, 모임 상세/참여)의 와이어프레임을 만든다.
2. 레벨 체계와 모임 템플릿의 문구를 확정한다.
3. 모임 데이터 모델과 상태값(모집중·마감·취소·완료)을 정의한다.
4. 첫 서비스 지역과 초기 호스트 모집 방식을 정한다.
5. 실제 테린이 인터뷰 및 수동 매칭 실험을 설계한다.
